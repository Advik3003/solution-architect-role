# Solution Architect

A **Solution Architect** designs end-to-end software solutions that are secure, scalable, maintainable, and aligned with business goals.

## Core Responsibilities

- Understand business requirements and convert them into technical architecture.
- Define system boundaries, integration patterns, and data flow.
- Select technologies, frameworks, and cloud services.
- Ensure non-functional requirements: scalability, performance, security, reliability, and observability.
- Guide development teams with architecture decisions and standards.

## Skills Needed (From Full-Stack to Solution Architect)

- Strong full-stack fundamentals (frontend, backend, database, APIs).
- System design and distributed systems understanding.
- Cloud architecture (AWS/Azure/GCP), IaC basics, CI/CD.
- Security basics (authentication, authorization, encryption, OWASP).
- Communication and stakeholder management.
- Cost-performance tradeoff analysis.

## Daily Outputs

- Architecture diagrams (context, container, component, sequence).
- API contracts and integration specs.
- Technology decision records (TDR/ADR).
- Capacity, risk, and security considerations.
- Implementation roadmap and team guidance.

## Career Shift Mindset

As a full-stack developer, you focus on implementing features.  
As a solution architect, you focus on **designing the whole system**, managing tradeoffs, and ensuring long-term business value.
